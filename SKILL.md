# writing-quality_diy — 中文写作质感增强

对标 Fable 5 文学性输出。通过风格约束 + 多轮迭代，消除「AI味」，实现自然流畅的专业中文写作。

---

## 何时使用

- 用户要求「把这段话润色一下」「让文字更有质感」
- 生成面向外部读者的文档（报告、方案、白皮书）
- 需要从草稿提炼出成品级文本
- 觉得输出「太像AI写的」的时候

---

## 两轮工作流

### 第一轮：生成草稿

如果用户提供了原始文字，先做风格评估再改；如果是从零写作，直接套用约束。

#### 风格评估（改写场景）

快速判断原文风格再动手：

| 维度 | 判断 | 改写方向 |
|------|------|---------|
| 语体 | 正式/半正式/口语 | 朝目标语体调整 |
| 句长 | 偏长/适中/偏碎 | 长句拆分或短句合并 |
| 节奏 | 单调/自然/跳跃 | 长短句交错 |
| 用词 | 翻译腔/书面语/口语 | 替换翻译腔和套话 |

#### 核心写作约束

写入时遵循以下规则：

```
## 写作约束（严格执行）

1. 句式：长短句交错，避免连续3句以上相同句式结构
2. 节奏：每段开头用短句建立锚点 → 中间展开论证 → 结尾回扣主题
3. 生动性：每500字至少1个具体例子或类比
4. 中国语境：使用中文读者熟悉的典故、成语、现象，避免翻译腔
5. 禁用词：综上所述、值得注意的是、不可否认、从某种意义来说、极大地、众所周知
6. 禁用句式："不仅……而且……"、"一方面……另一方面……"
7. 段落长度：每段3-5句，不超过150字
8. 标题：用动词引领，避免名词堆砌
```

### 第二轮：自查优化

生成后自检以下清单，逐项修正：

```
## 自检清单

□ 有没有连续3句以上都是相同句式？（如连续3句"XX是YY"）
□ 有没有「综上所述」「值得注意的是」等套话？
□ 每个段落是否都控制在150字以内？
□ 每500字有没有至少一个具体例子？
□ 开头有没有用一个有力的短句抓住注意力？
□ 结尾有没有回扣主题而不是重复？
□ 读起来像人写的吗？（闭眼默读一遍）
```

如需第三轮，再跑一次第二轮的检查。

---

## 不适用场景

- 纯信息传递（查资料、快速问答）——保持简洁优先
- 代码和命令行输出——不改自然语言规则
- 翻译任务——用 cn2en_med_diy / en2cn_diy
- 学术论文——保持学术规范高于文学性

---

## 示例

用户输入：
> AI大模型正在改变软件开发的方式。

输出草稿：
> 一个模型，吃掉百万行代码，吐出一个可运行的系统——这不是未来，是 Fable 5 上周干的事。Stripe 把五千万行 Ruby 交给它，一天跑完。而三个月前，同样的事需要两个工程师干一年。这不是效率提升了几个百分点，是范式坍塌了。软件开发从「写代码」变成「描述需求」，从「调试」变成「审查」。不是每个团队都准备好了，但准备不准备的，门槛已经碎了。
